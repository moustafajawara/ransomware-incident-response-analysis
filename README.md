# Ransomware Incident Response Analysis

This project analyzes a simulated ransomware attack conducted during a cybersecurity tabletop exercise.

## Overview
The incident began with a successful phishing email that compromised a remote user’s system. The attacker laterally moved through the network and encrypted critical business data using AES-256 and RSA-2048 encryption.

The organization experienced significant operational disruption including system downtime, productivity loss, and financial impact.

## Key Topics Covered
- Ransomware attack lifecycle
- Phishing-based initial compromise
- Lateral movement within enterprise networks
- Incident response decision making
- Business continuity planning

## Security Controls Evaluated
- Multi-Factor Authentication (MFA)
- Network Segmentation
- VPN Access Controls
- IDS/IPS Monitoring
- Backup and Warm-Site Recovery

## Frameworks Used
- NIST Cybersecurity Framework
- Defense-in-Depth Security Model
- Maximum Tolerable Downtime (MTD)
- Recovery Time Objective (RTO)

## Key Findings
The organization was able to recover within 36 hours, staying below the maximum tolerable downtime of 72 hours. Backup systems and network segmentation significantly reduced the impact of the attack.
